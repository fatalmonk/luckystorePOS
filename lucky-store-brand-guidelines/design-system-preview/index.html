Lucky Store POS Design System v2

Category: Retail POS · Omnichannel · Bangladesh

Operational design language for neighborhood retail. Two surfaces (Storefront + POS), one token source. Built for cashiers, inventory staff, supervisors, and customers.

Architecture

The system is split into two distinct surfaces sharing one token foundation:

  • Section A — Customer Storefront: Large cards, imagery, wishlist, promotions. Optimized for browsing on mobile/desktop.
  • Section B — POS Operations: Compact cards, dense tables, checkout panel, barcode scanner, payments, receipts. Optimized for speed and all-day countertop use.

This separation prevents the classic mistake of forcing catalog UI patterns onto high-frequency POS workflows.

Color System

Warm minimal palette: ivory paper (#faf8f5), white surfaces, yellow accent (#FFF34D). No cold blue-greys. Semantic pairs (text + bg) for Success/Warning/Danger. Bangladesh payment method colors: Cash (green), bKash (red), Nagad (magenta), Card (blue).

Dark mode tokens defined but not yet activated — toggle in top-right to preview.

Typography

Inter (Latin) + Hind Siliguri (Bengali) — fluid clamp() scale, no fixed px breakpoints. Price uses H2 scale at weight 700. Mono uses JetBrains Mono for SKUs, prices, timestamps.

Spacing & Radius

4px base unit (space-1 through space-16). Four radius sizes with radius-md (12px) as universal default. Shadows use warm black tint (rgba(28,25,23,α)) for cohesive elevation.

Motion

Four standardized durations (80/150/200/300ms) with cubic-bezier(0.4,0,0.2,1). Active press scale(0.97), hover lift translateY(-1px). Full prefers-reduced-motion support.

POS-Specific Components

POS Product Card

56×56px thumbnail, SKU, Cost/MRP/Sell prices, margin %, stock, qty stepper. One line — built for scanning speed, not browsing.

Checkout Panel

Invoice header, line items (name, qty, price), running totals (subtotal, discount, VAT, grand total), payment method buttons (Cash, bKash, Nagad, Card, Split). The operational heart.

Payment States

Four order states (Awaiting, Paid, Partial, Failed) + four method badges (Cash, bKash, Nagad, Card) + method selector buttons.

Inventory Table

Sticky headers, zebra stripes, hover highlight, mobile horizontal scroll. SKU (mono), Product (truncated), Cost/Sell (mono, right), Margin (mono, right, green), Stock (mono, right, red/yellow/green), Actions.

Receipt Templates

58mm (standard) and 80mm (detailed) thermal formats. Monospace, dashed separators, QR placeholder. ESC/POS ready.

Barcode Scanner

Three states: Scanning (animated), Success (product data), Error (create/manual fallback). Full-screen overlay for hardware integration.

Empty States

No Products, No Orders, Offline Mode, Empty Inventory — all with icon, title, description, primary action.

Loading Skeletons

Product card, table rows, checkout panel. Shimmer animation (1.5s) respects prefers-reduced-motion.

Dashboard KPIs

Six cards: Today's Sales, Gross Margin, Low Stock, Transactions, Cash in Drawer, Digital Payments %. Trend indicators (positive/negative/neutral) with sparkline-ready SVG arrows.

Cross-Platform Token Sync

Canonical source: design-tokens/tokens.json. Build outputs:

  • output/tokens.css → apps/customer_storefront/app/styles/tokens.css
  • output/tokens.ts → web apps
  • output/tokens.dart → apps/mobile_app/lib/core/theme/tokens.dart

Run npm run build in design-tokens/ after edits. Tailwind mapping uses warm-* prefix (e.g., bg-warm-bg → var(--color-paper)).

Accessibility

  • Contrast: 4.5:1 body text (#1c1917 on #faf8f5), 3:1 large text on accent
  • Focus ring: 2px solid var(--color-accent) with 2px offset
  • Tap targets: minimum 44×44px (all buttons, inputs, clickable rows)
  • prefers-reduced-motion: transitions disabled
  • Dark mode tokens ensure future AA compliance
